# Spherepop  

[Audio Lectures](https://standardgalactic.github.io/spherepop/projects/)

### A Calculus of History, Irreversible Commitment, and World Construction

Spherepop is a computational framework and philosophical calculus that redefines identity as **causal history** rather than current state.

Instead of asking *“What is the system now?”*, Spherepop asks:

> **“What sequence of irreversible choices created this system?”**

This project explores a model of computation where:

- Time is irreversible  
- Choices destroy alternatives  
- History is the primary object  
- State is reconstructed, not stored  

---

## Core Idea

A Spherepop system is defined by:

    (Ω_t, H_t)

- **Ω_t (Option Space)** — remaining possible futures  
- **H_t (History)** — ordered sequence of past commitments  

### Fundamental Law

- Ω_t shrinks over time  
- H_t grows over time  

> To move forward is to lose options.  
> To exist is to accumulate history.

---

## Why Spherepop?

Modern computation assumes:

- State can be overwritten  
- Actions can be undone  
- History is disposable  

Spherepop rejects this.

### Instead:

- No undo  
- No reset  
- No collapsing of paths  

Two systems with identical outputs but different histories are **not the same system**.

---

## Key Operators

Spherepop is built from a small set of primitives:

| Operator | Function |
|----------|---------|
| **Pop** | Irreversible choice (removes an option, records it) |
| **Bind** | Constraint (filters options without recording) |
| **Collapse** | Reconstructs state by replaying history |
| **Refuse** | Ethical exclusion (intentional non-choice) |

---

## Key Insights

### 1. Identity is Historical

The path taken defines the system, not the final result.

---

### 2. Reality is Reconstructed

There is no stored state.

    state = replay(history)

---

### 3. Choice is Destructive

Every commitment removes alternative futures permanently.

---

### 4. Meaning Requires Loss

A system that never commits never becomes anything.
