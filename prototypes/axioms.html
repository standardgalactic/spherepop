Axioms and Theorems

Spherepop OS is defined by a small number of axioms. All system properties follow as theorems or corollaries.

This page states the core axioms explicitly and records the principal results they entail.

Ontology

Axiom O1 (Semantic Objects)

A semantic object is an abstract handle with no intrinsic meaning. Objects acquire meaning only through relations, equivalence, and the event history that produced them.

Axiom O2 (Relational Primacy)

All semantic structure arises from typed relations between objects and from equivalence relations induced by events.

Corollary O2.1

There exists no authoritative object table independent of replay. Objects exist only as derived consequences of the event log.

Authority and Time

Axiom T1 (Append-Only Authority)

All authoritative semantic state transitions are recorded as events in a single append-only log. No other mechanism may introduce or modify authoritative state.

Axiom T2 (Total Causal Order)

Every event in the log is assigned a unique position in a total order. This order is global and unambiguous.

Theorem T2.1 (Well-Defined Time)

Semantic time in Spherepop OS is isomorphic to log prefix order. There is no notion of “current state” independent of a prefix.

Corollary T2.2

Two observers referring to the same log prefix necessarily refer to the same semantic state, regardless of transport or representation.

Kernel Semantics

Axiom K1 (Deterministic Kernel)

The kernel is a deterministic transition system. For any derived state σ and event e, there exists at most one σ′ such that σ —e→ σ′.

Axiom K2 (No Hidden State)

The kernel maintains no hidden or mutable state beyond that derivable from replay of the event log.

Theorem K2.1 (Deterministic Replay)

Given an initial empty state and a log prefix ℓ, replay yields a unique derived state σℓ.

Corollary K2.2

Any cache, index, or materialized view may be discarded and reconstructed without semantic loss.

Equivalence and Canonicalization

Axiom E1 (Event-Induced Equivalence)

Equivalence between objects is induced exclusively by MERGE or COLLAPSE events.

Axiom E2 (Representative Normalization)

All relations are stored and interpreted in representative-normalized form with respect to the induced equivalence relation.

Theorem E2.1 (Merge Confluence)

The final equivalence relation induced by a set of MERGE events is independent of their order.

Corollary E2.2

Canonical representatives are stable under replay and transport.

Views and Non-Interference

Axiom V1 (View–Cause Separation)

Views are derived representations of kernel state and may not introduce or modify authoritative events.

Axiom V2 (Non-Interference)

Dropping, reordering, or recomputing views does not affect authoritative semantics.

Theorem V2.1 (Snapshot Purity)

A snapshot introduces no information not already present in the log prefix it represents.

Theorem V2.2 (Replay Equivalence)

Incremental observation via diffs is semantically equivalent to full replay from the log prefix.

Functorial Semantics

Axiom F1 (Prefix Category)

Log prefixes form a category whose objects are prefix lengths and whose morphisms are prefix extensions.

Axiom F2 (State Semantics Functor)

Replay defines a functor from the prefix category into the category of kernel states.

Theorem F2.1 (Functoriality of Replay)

Replay respects identity and composition. Sequential event application corresponds to morphism composition.

Theorem F2.2 (Functorial Views)

Any admissible view factors functorially through replay. View updates therefore respect causal order by construction.

Speculation

Axiom S1 (Speculative Isolation)

Speculative events exist only in local overlay logs and have no effect on authoritative state unless explicitly committed.

Theorem S1.1

Speculation cannot corrupt or destabilize the authoritative timeline.

Corollary S1.2

Exploratory reasoning and counterfactual analysis are safe operations.

Geometry and Metadata

Axiom G1 (Geometry as Metadata)

Layout, position, and visualization hints are metadata and are not interpreted as semantic constraints.

Theorem G1.1 (Gauge Freedom)

Two views differing only in geometric metadata are semantically equivalent.

Summary

Spherepop OS is not defined by features but by invariants. Its axioms enforce determinism, causal clarity, and semantic stability. Everything else—tools, interfaces, utilities—is a derived consequence.
